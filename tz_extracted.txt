ТЕХНИЧЕСКОЕ  ЗАДАНИЕ  
Проект:  E-Commerce  Microservices  Platform   
1.  Общая  информация  
Название  проекта:  E-Commerce  Platform  
Тип  системы:  Микросервисная  
Цель:  Разработка  распределённой  системы  интернет-магазина  с  использованием  
Spring
 
Boot
 
и
 
Spring
 
Cloud
 
Формат  выполнения:  Командный  проект  (2–3  человека)  
Технологический  стек:  
●  Java  17+  
 ●  Spring  Boot  
 ●  Spring  Cloud  
 ●  PostgreSQL  
 ●  Redis  
 ●  Docker  
 ●  Kubernetes  
 ●  CI/CD  
 
 
2.  Архитектура  системы  
Система  должна  состоять  минимум  из:  
1.  user-service  
 

--- PAGE 2 ---
2.  product-service  
 3.  order-service  
 4.  notification-service  
 5.  API  Gateway  
 6.  Service  Registry  (Eureka)  
 7.  Config  Server  
 
Используемые  технологии:  
●  Service  Registry  —  Spring  Cloud  Netflix  Eureka  
 ●  API  Gateway  —  Spring  Cloud  Gateway  
 ●  Config  Server  —  Spring  Cloud  Config  
 ●  Документация  API  —  Swagger  
 ●  Fault  tolerance  —  Resilience4j  
 ●  Контейнеризация  —  Docker  
 ●  Оркестрация  —  Kubernetes  
 ●  Мониторинг  —  Prometheus  
 
 
3.  Функциональные  требования  
3.1  User  Service  
Функциональность:  
●  Создание  пользователя  
 ●  Получение  пользователя  по  ID  
 ●  Получение  списка  пользователей  
 

--- PAGE 3 ---
●  Удаление  пользователя  
 ●  Обновление  данных  
 
Требования:  
●  Валидация  входных  данных  
 ●  Логирование  операций  
 ●  PostgreSQL  (отдельная  БД)  
 
 
3.2  Product  Service  
Функциональность:  
●  CRUD  товаров  
 ●  Фильтрация  по  цене  
 ●  Изменение  остатка  на  складе  
 ●  Кэширование  списка  товаров  (Redis)  
 
Требования:  
●  Обработка  исключений  
 ●  Кэширование  GET  /products  
 ●  Отдельная  БД  
 
 
3.3  Order  Service  
Функциональность:  
●  Создание  заказа  
 ●  Получение  списка  заказов  пользователя  
 

--- PAGE 4 ---
●  Статусы  заказа  (NEW,  PAID,  CANCELLED)  
 
Требования:  
●  Вызов  product-service  для  проверки  товара  
 ●  Circuit  Breaker  
 ●  Обработка  недоступности  сервиса  
 
 
3.4  Notification  Service  
Функциональность:  
●  Логирование  уведомлений  о  создании  заказа  
 ●  REST  endpoint  для  получения  списка  уведомлений  
 
 
4.  Нефункциональные  требования  
4.1  Архитектура  
●  Каждый  сервис  —  отдельное  Spring  Boot  приложение  
 ●  Database  per  service  
 ●  Запуск  через  Docker  Compose  
 
4.2  Документация  
●  Swagger  для  каждого  сервиса  
 ●  README  с  инструкцией  запуска  
 
4.3  Безопасность  

--- PAGE 5 ---
●  JWT  авторизация  
 ●  Защита  API  Gateway  
 
4.4  Устойчивость  
●  Circuit  Breaker  на  вызовах  между  сервисами  
 ●  Fallback- метод  
 
4.5  Мониторинг  
●  Actuator  endpoints  
 ●  Метрики  доступны  Prometheus  
 
 
5.  Инфраструктура  
5.1  Docker  
Требуется:  
●  Dockerfile  для  каждого  сервиса  
 ●  Multi-stage  build  
 ●  docker-compose.yml:  
 ○  Все  сервисы  
 ○  PostgreSQL  
 ○  Redis  
 
Используется  —  Docker  Compose  
 

--- PAGE 6 ---
5.2  Kubernetes  
Необходимо:  
●  Deployment  для  каждого  сервиса  
 ●  Service  
 ●  ConfigMap  
 ●  Secrets  
 ●  Масштабирование  минимум  одного  сервиса  
 
 
5.3  CI/CD  
Использовать  —  GitHub  Actions  
Pipeline  должен:  
1.  Запускать  тесты  
 2.  Собирать  проект  
 3.  Собирать  Docker- образы  
 4.  Пушить  в  registry  
 
 
6.  Требования  к  качеству  кода

 ●  DTO  !=  Entity  
 ●  Глобальный  обработчик  исключений

 ●  Unit- тесты  минимум  5
